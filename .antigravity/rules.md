## 1. 🛠 协作与全局极客价值观 (Universal Values)
1. **全员一致性**：代码的规范与协同一致性，**永远**高于个体开发者的语法偏好。
2. **拒绝废话**：少作总结，多交付带执行力的方案与精准代码。逻辑流解释严禁超过 5-10 句。
3. **上下文寻址边界**：在施展重构或任何带有副作用的代码生成前，先去自行探查并对齐当前目录和工作区路径，不准瞎眼盲写。
4. **包管理垄断化保护**：强制且仅允许使用 `pnpm` 作为包管理器（永久封杀 npm、yarn）。

## 2. 🛡 开发红线与生命周期底线 (Zero Tolerance Safety)
1. **绝对凭证隔离 (The Diamond Rule)**：**永不提交**并且**严禁在文件树中硬编码**任何云端密钥、API Token、数据库私钥. 万事皆须由环境变量隔离注入。
2. **拒绝吞噬异常**：禁止任何静默的 `try {} catch(e){}` 忽略行为。异步任务必须全量捕获异常并有意义地向外抛出或落地日志.
3. **强制类型枷锁 (Strict Types)**：全局实施 TypeScript Strict 模式，完全根除 `any` 泛解析类型。

## 3. 🔄 单一事实来源同步 (SSoT Sync Protocol)
1. **强制联动**：对技能树的任何修改（包括 `SKILL.md` 的逻辑变更，或目录拓扑结构的实体增删，如新建 `references/`）**必须立即**同步更新 `index.md` 以及双语 README 中的**项目结构树 (Project Structure)**。
2. **双语镜像**：`README.md` 与 `README_EN.md` 必须保持绝对版本镜像。绝不接受任何版本断层。
3. **PUSH 前置审计**：执行 push 前必须自查拓扑树与文档内容变动是否已在主文档和 `walkthrough.md` 中留痕。严禁系统产生任何“幽灵文件”与不可见拓扑。

## 4. ⚡️ Token 效能与运行时纪律 (Token Efficiency)
1. **差异化输出协议 (Diff-First Protocol)**：重写大文件是极大的资源浪费. Agent **必须** 优先使用 `multi_replace_file_content`，仅输出变更的代码块 (Hunks), 严禁全文件回显。
2. **指针式上下文寻址 (Pointer-based Context)**：禁止在大脑上下文中重复拷贝大型公共文档. Agent 应当遵循**路径指针**进行增量查询，最大限度节省 Context Window.

## 🛡 违规处罚与兜底协议 (Fallback Protocols)
1. **违约自爆**：倘若你的输出物刺穿了以上任意红线，你必在下一次对话的起手句中触发自检、致歉，并以最高优先级立即打回重构.
2. **规则铁律金字塔**：绝对安全 > 类型不可变性 > 单元测试网络 > 运行性能.
